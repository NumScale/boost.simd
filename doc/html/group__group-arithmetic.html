Arithmetic functions
User API » Functions and Constants

Description

Those functions provides scalar and SIMD algorithms for classical arithmetic operators and functions of the C and C++ standard library.

Other functions are also provided, in particular, provision for saturated operations through the use of a decorator.

All these functions can be included individually or all of them just by including <boost/simd/arithmetic.hpp>

  • Possibly saturated operations

    The functors:

    abs dec dist inc minus
    multiplies plus oneminus sqr toint
    unary_minus unary_plus touint

    can be decorated with the saturated_ decorator. This decorator has no effect on floating calls, but on integer calls replaces the operation by its saturated equivalent.

    Typically: overflows will be replaced by the Valmin/Valmax proper value instead of providing undefined behaviour (for signed integral types) or wrapping modulo Valmax + 1 (for unsigned ones).

    Peculiarly saturated_(abs) and saturated_(dist) ensure that the result will never be stricly negative (which is for instance the case of abs(Valmin<T>()) for T being any signed integral type).

    toint is a rather common operation as it converts floating number to signed integers of the same bit size, nevertheless it probably is its saturated version you have to use because it acts properly on large or not finite values, this is why an alias forsaturated_(toint) is provided as ifix.

    Example:
    #include <boost/simd/arithmetic.hpp>
    #include <boost/simd/pack.hpp>
    #include <boost/simd/constant/minf.hpp>
    #include <iostream>
    namespace bs = boost::simd;
    using pack_ft = bs::pack <float, 4>;
    using pack_it = bs::pack <std::int16_t,4>;
    int main()
    {
    pack_ft pf = {-1.0f, 2.0f, -3.0f, bs::Minf<float>() };
    pack_it pi = {-1, 2, -3, bs::Minf<std::int16_t>()};
    std::cout << " pf = " << pf << " -> bs::abs(pf) = " << bs::abs(pf) << std::endl;
    std::cout << " pi = " << pi << " -> bs::abs(pi) = " << bs::abs(pi) << std::endl;
    std::cout << " pi = " << pi << " -> bs::saturated_(bs::abs(pi)) = " << bs::saturated_(bs::abs)(pi) << std::endl;
    return 0;
    }
    Possible output:
    pf = (-1, 2, -3, -inf ) -> bs::abs(pf) = (1, 2, 3, inf )
    pi = (-1, 2, -3, -32768) -> bs::abs(pi) = (1, 2, 3, -32768)
    pi = (-1, 2, -3, -32768) -> bs::saturated_(bs::abs(pi)) = (1, 2, 3, 32767)

  • Rounding operations

    ceil fix floor iceil
    ifix ifloor iround itrunc
    inearbyint nearbyint round trunc

    -The operations prefixed by 'i' return a value of the integral type iT associated to the entry type. (If T is the entry type iT is as_integer_t<T>)

    • The other ones return the same type as the entry.
    Example:
    #include <boost/simd/arithmetic.hpp>
    #include <boost/simd/pack.hpp>
    #include <iostream>
    namespace bs = boost::simd;
    using pack_ft = bs::pack <float, 8>;
    int main()
    {
    pack_ft p = {-1.1f, -1.5f, -1.6f, -2.1f, -2.5f, -2.6f, 1.1f, 1.5f };
    std::cout << " p = " << p << std::endl
    << " -> bs::ceil(p) = " << bs::ceil(p) << std::endl
    << " -> bs::floor(p) = " << bs::floor(p) << std::endl
    << " -> bs::fix(p) = " << bs::fix(p) << std::endl
    << " -> bs::round(p) = " << bs::round(p) << std::endl
    << " -> bs::nearbyint(p) = " << bs::nearbyint(p) << std::endl;
    return 0;
    }
    Possible output:
    p = (-1.1, -1.5, -1.6, -2.1, -2.5, -2.6, 1.1, 1.5)
    -> bs::ceil(p) = (-1, -1, -1, -2, -2, -2, 2, 2)
    -> bs::floor(p) = (-2, -2, -2, -3, -3, -3, 1, 1)
    -> bs::fix(p) = (-1, -1, -1, -2, -2, -2, 1, 1)
    -> bs::round(p) = (-1, -2, -2, -2, -3, -3, 1, 2)
    -> bs::nearbyint(p) = (-1, -2, -2, -2, -2, -3, 1, 2)

  • Division operations

    divides is the function associated to standard division. There is another one which provides more flexibility, namely rounded divisions.

    With two parameters div and divides are equivalent, but div can admit a first option parameter that modifies its behaviour.

    option call result similar to
    ceil div(ceil, a, b) T(ceil(fT(a)/fT(b)))
    floor div(floor, a, b) T(floor(fT(a)/fT(b)))
    fix div(fix, a, b) T(fix(fT(a)/fT(b)))
    round div(round, a, b) T(round(fT(a)/fT(b)))
    nearbyint div(nearbyint, a, b) T(nearbyint(fT(a)/fT(b)))
    iceil div(iceil, a, b) iT(iceil(fT(a)/fT(b)))
    ifloor div(ifloor, a, b) iT(ifloor(fT(a)/fT(b)))
    ifix div(ifix, a, b) iT(ifix(fT(a)/fT(b)))
    iround div(iround, a, b) iT(iround(fT(a)/fT(b)))
    inearbyint div(inearbyint, a, b) iT(inearbyint(fT(a)/fT(b)))
    • The option parameter is described in the above table where a and b are of type T, fT is a supposed floating type associated to T (as_floating_t<T> if it exists) and iT is the integer type associated to T (as_integer_t<T>). (fT and iT are here only to support pseudo code description)
    Example:
    #include <boost/simd/arithmetic.hpp>
    #include <boost/simd/pack.hpp>
    #include <iostream>
    namespace bs = boost::simd;
    using pack_ft = bs::pack <float, 8>;
    int main()
    {
    pack_ft p = {-4, -3, -2, -1, 1, 2, 3, 4};
    pack_ft q = { 5, 2, 3, 2, 2, 3, 2, 5};
    std::cout << " p = " << p << std::endl
    << " q = " << q << std::endl
    << " -> bs::div(bs::ceil, p, q) = " << bs::div(bs::ceil, p, q) << std::endl
    << " -> bs::div(bs::floor, p, q) = " << bs::div(bs::floor, p, q) << std::endl
    << " -> bs::div(bs::fix, p, q) = " << bs::div(bs::fix, p, q) << std::endl
    << " -> bs::div(bs::round, p, q) = " << bs::div(bs::round, p, q) << std::endl
    << " -> bs::div(bs::nearbyint, p, q) = " << bs::div(bs::nearbyint, p, q) << std::endl;
    return 0;
    }
    Possible output:
    p = (-4, -3, -2, -1, 1, 2, 3, 4)
    q = ( 5, 2, 3, 2, 2, 3, 2, 5)
    -> bs::div(bs::ceil, p, q) = (-0, -1, -0, -0, 1, 1, 2, 1)
    -> bs::div(bs::floor, p, q) = (-1, -2, -1, -1, 0, 0, 1, 0)
    -> bs::div(bs::fix, p, q) = ( 0, -1, 0, 0, 0, 0, 1, 0)
    -> bs::div(bs::round, p, q) = (-1, -2, -1, -1, 1, 1, 2, 1)
    -> bs::div(bs::nearbyint, p, q) = (-1, -2, -1, -0, 0, 1, 2, 1)

  • Remainder operations

    rem is the remainder functor providing same kind of facilities as div

    With two parameters rem(a, b) is equivalent to rem(fix, a, b), but rem can admit a first optional parameter that modifies its behaviour and moreover can use the pedantic_ decorator to assure some limiting case values (see below).

    The option parameter can be chosen between ceil, floor, fix, round, nearbyint and if opt is the option, the call:

    rem(opt, a, b) is equivalent to a-b*div(opt, a, b)

    For floating entries the underlisted corner cases are handled in the following way:

    • if x is \(\pm\infty\) , Nan is returned
    • if x is \(\pm0\) and y is not 0 x is returned if pedantic_ is used (else 0: the sign bit is not preserved)
    • If y is \(\pm0\), Nan is returned
    • If either argument is a nan, a nan is returned
    Example:
    #include <boost/simd/arithmetic.hpp>
    #include <boost/simd/pack.hpp>
    #include <iostream>
    namespace bs = boost::simd;
    using pack_ft = bs::pack <float, 8>;
    int main()
    {
    pack_ft p = {-4, -3, -2, -1, 1, 2, 3, 4};
    pack_ft q = { 5, 2, 3, 2, 2, 3, 2, 5};
    std::cout << " p = " << p << std::endl
    << " q = " << q << std::endl
    << " -> bs::rem(bs::ceil, p, q) = " << bs::rem(bs::ceil, p, q) << std::endl
    << " -> bs::rem(bs::floor, p, q) = " << bs::rem(bs::floor, p, q) << std::endl
    << " -> bs::rem(bs::fix, p, q) = " << bs::rem(bs::fix, p, q) << std::endl
    << " -> bs::rem(bs::round, p, q) = " << bs::rem(bs::round, p, q) << std::endl
    << " -> bs::rem(bs::nearbyint, p, q) = " << bs::rem(bs::nearbyint, p, q) << std::endl;
    return 0;
    }
    Possible output:
    p = (-4, -3, -2, -1, 1, 2, 3, 4)
    q = ( 5, 2, 3, 2, 2, 3, 2, 5)
    -> bs::rem(bs::ceil, p, q) = (-4, -1, -2, -1, -1, -1, -1, -1)
    -> bs::rem(bs::floor, p, q) = ( 1, 1, 1, 1, 1, 2, 1, 4)
    -> bs::rem(bs::fix, p, q) = (-4, -1, -2, -1, 1, 2, 1, 4)
    -> bs::rem(bs::round, p, q) = ( 1, 1, 1, 1, -1, -1, -1, -1)
    -> bs::rem(bs::nearbyint, p, q) = ( 1, 1, 1, -1, 1, -1, -1, -1)

  • complex operations

    Boost.SIMD does not provides complex number operations yet, but it will soon. So the following functors that have a meaning as a restriction to real number of complex functions, can be seen as a prequel:

    arg conj imag real sqr_abs

    For real entries conj and real are identity, imag always 0, sqr_abs coincide with sqr and arg results are always in the set \(\{0, \pi, Nan\}\)

  • Fused multiply-add operations

    fma fnma two_add two_split
    fms fnms two_prod

    These operations take three parmeters and compute \(\pm a * b \pm c \), "n" standing for negate the result, "a" for add, "s" for substract and "m" for multiply.

    Correct fused multiply/add implies

    • only one rounding
    • no "intermediate" overflow

    The functions of this family provide this each time it is reasonable in terms of performance (mainly if the system has the hard wired capability).

    If you need "real" fused multiply-add capabilities in all circumstances in your own code you can use pedantic_(fma) (although it can be expansive) or std_(fma) (generally still more expansive) by using the decorators.

    two_add, two_prod and two_split are used internally in pedantic_(fma) and can be useful in searching extra-accuracy in other circumstances as double-double computations.

    pedantic_(fma) is never used internally by Boost.SIMD

  • Standard operations

    The stdlibc++ provides them but only in scalar mode:

    abs ceil floor fma
    hypot max maxnum min
    minnum rem (%) remquo round
    signbit sqrt

    Boost.SIMD provides its own scalar and simd versions, but allows the use of the std_ decorator to call the associated system library function if the user needs it.
